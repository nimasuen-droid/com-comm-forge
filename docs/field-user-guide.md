# Field User Guide

## Audience

Field engineers, discipline leads, punch coordinators, preservation teams, commissioning technicians, and vendor representatives.

## Field Workflow

1. Open the project before going offline.
2. Confirm the correct system/subsystem.
3. Capture or update the field record.
4. Save locally.
5. Reconnect and confirm pending work remains visible.
6. Notify the completions lead if conflicting information is found.

## Systems

Use Systems to understand the work boundary.

Check:

- System code.
- Subsystem code.
- Discipline.
- Tags.
- MC/RFSU/Commissioning/Turnover status.

Do not rename system/subsystem codes in the field unless instructed by the completions lead.

## Preservation

Use Preservation to keep equipment protected before commissioning.

Steps:

1. Find subsystem/equipment.
2. Confirm preservation interval.
3. Enter last-done date.
4. Review next due date and overdue status.
5. Save.

Escalate overdue or missing preservation intervals.

## Punch List

Use Punch to raise and close field defects.

Punch categories:

- A: Blocks MC/RFSU or safe startup.
- B: Must close before handover or agreed milestone.
- C: Non-blocking closeout item.

Raise a punch with:

- Clear title.
- Description of defect.
- Category.
- Discipline.
- Responsible party.
- System/subsystem.
- Due date when known.
- Evidence where available.

Do not downgrade Cat A punches without discipline lead approval.

## Mechanical Completion

Use MC for static completion readiness.

Only tick checklist items when the field condition is verified:

- Walkdown signed.
- Hydrotest pack closed.
- Flushing/blowing complete.
- Reinstatement complete.
- Preservation acceptable.
- Cat A punches cleared.

## Commissioning

Use Commissioning for dynamic readiness.

Check sequence:

1. Energisation.
2. Loop checks.
3. Cause and Effect.
4. Functional testing.
5. Performance testing.
6. Reliability run.

Do not advance commissioning checks when MC is incomplete or Cat A punches remain open.

## Turnover

Use Turnover when preparing for operations acceptance.

Confirm:

- MC certificate.
- RFSU certificate.
- Commissioning complete.
- Operations acceptance.
- Care, Custody, and Control transfer.

## Offline Work

Before going offline:

- Open the project and relevant module while online.
- Confirm data is visible.

While offline:

- Continue edits.
- Save locally.
- Avoid duplicate work with another user if possible.

After reconnecting:

- Refresh only after confirming saved work is visible.
- Notify admin if sync/conflict indicators appear.

## Field Quality Rules

- Use precise subsystem references.
- Write punch descriptions that another person can execute.
- Add dates in local project timezone.
- Keep comments professional and factual.
- Never delete records to hide field history.
